# 应届生求职方向介绍：嵌入式、AI 与它们的交叉地带

> 这份文档给徐超峰：在投简历之前，先想清楚一个问题——**「基于STM32的智能安全插座」这个毕设，到底该往哪个岗位方向去包装？**
>
> 同一个项目，面向不同岗位要强调的东西完全不同。投嵌入式岗，HR 想看你会不会画 PCB、会不会写中断；投 AI 岗，想看你懂不懂 LLM Agent 的设计。先选方向，再决定简历重心，而不是写一份通用简历海投。

---

## 先看结论：这个毕设天然适配哪些方向

这个项目的结构其实很特别——它是一个 **「端云协同」** 系统：

- **边缘端（STM32）**：实时采集、本地显示、继电器通断、过载/漏电的硬中断保护——这是**典型的嵌入式开发**。
- **云端（OpenClaw + LLM Agent）**：自然语言入口、意图理解、多步规划、冲突检测、场景编排——这是**典型的 AI 应用 / Agent 开发**。
- **中间（ESP8266 + MQTT）**：联网上报与指令下行——这是**物联网（IoT）**。

也就是说，一个毕设同时踩在了三条线上。这是优势，但也意味着你**不能三条线平均用力**——简历篇幅有限，平均用力 = 每一条都不深。下面把四个方向摊开对比。

---

## 四个方向横向对比

| 方向 | 核心岗位 | 关键技能栈 | 市场情况 | 本毕设能支撑的部分 |
|------|---------|-----------|---------|------------------|
| **嵌入式 / 硬件开发** | 嵌入式软件工程师、单片机开发、硬件工程师 | C、STM32/ARM Cortex-M、寄存器/外设（ADC/UART/IIC）、中断与定时器、RTOS、电路与 PCB、示波器调试 | 需求稳定、岗位多，工业控制/消费电子/汽车电子都要人；门槛实打实，但天花板和涨薪偏慢 | **最扎实的一块**：STM32 主控、电流采集+滑动平均、继电器+光耦隔离驱动、过载/漏电中断保护、DS1302/OLED 外设、PCB 模块化分区 |
| **AI 应用 / Agent** | AI 应用工程师、LLM 应用开发、Agent 开发、Prompt 工程 | Python、LLM API、Prompt Engineering、RAG、Agent 架构（ReAct/多步规划）、MCP、工具/Skill 设计 | 最热、薪资高、扩张快；但竞争者背景普遍偏「算法/CS 科班 + 项目」，纯嵌入式背景需要补 Python 工程与 LLM 基础 | **决策层这一块**：OpenClaw 消息桥接 + Skill 容器、LLM Agent 意图理解/多步规划/冲突检测/场景编排、MCP 风格原子能力、安全护栏 |
| **嵌入式 + AI 复合**（推荐） | 边缘 AI、AIoT 工程师、智能硬件开发、具身/机器人方向的底层 | 上面两栏的交集 + 「端云协同」系统思维：什么放边缘（实时、安全）、什么放云端（推理、规划） | 数量比纯嵌入式少，但**这正是它的价值**——会硬件的人多、懂 LLM 的人多，两边都能接上的人稀缺，溢价明显 | **整个项目**：从 STM32 边缘实时控制，到 ESP8266 联网，再到云端 LLM Agent，一条完整的端到端链路你都走过 |
| **物联网 / IoT** | IoT 开发、智能家居、嵌入式联网 | 嵌入式基础 + 通信协议（MQTT/Wi-Fi/BLE）、云平台、移动端联调 | 介于嵌入式和云之间，智能家居/工业物联网有稳定需求 | **通信链路这一块**：ESP8266 Wi-Fi、MQTT Broker、移动端 APP 交互、端云数据闭环 |

---

## 为什么推荐「嵌入式 + AI 复合」

**先说约束**：你是光电专业的本科应届生，最硬的筹码就是这个毕设；它最大的特点不是「嵌入式做得多深」，也不是「AI 模型多先进」，而是**两端被你用一套系统串起来了**。

如果只投纯嵌入式岗，AI 那半截浪费了，而且和电子/自动化科班的嵌入式选手比，你的硬件深度未必占优；如果只投纯 AI 岗，硬件那半截浪费了，而且和 CS 科班的 Agent 选手比，你的 Python/LLM 工程经验偏薄。

**两边各砍一半都吃亏，但合起来反而独特**——市场上「会写 STM32 中断、又理解 LLM Agent 怎么规划任务」的应届生很少。当下边缘 AI、AIoT、智能硬件、机器人都在招这种能「上下打通」的人。

**适合什么样的人**：愿意同时碰硬件和软件、不想被锁死在单一技术栈、对「系统怎么协同」比「单点怎么极致优化」更有兴趣的人。如果你恰恰享受把毕设两端打通的过程，这个方向就是你的真实优势所在。

**简历里怎么体现两条线**（关键）：
- **核心优势**写两条——一条「嵌入式全链路」（带硬指标：保护动作 4.2ms、计量误差 1.27%），一条「端云协同 AI 能力」（OpenClaw + LLM Agent）。
- **项目经历**按「硬件设计 → 边缘软件 → AI 决策层 → 系统测试」分层展开，让读者看到你对**哪层该解决什么问题**有清晰判断——这正是复合方向最想看到的「系统思维」。
- **技能清单**分组列：嵌入式 / 通信 IoT / AI-Agent，让筛简历的人一眼看到你的技能横跨两端。

## 给应届生的选择建议

| 如果你… | 建议方向 | 后续要补的能力 |
|---------|---------|--------------|
| 喜欢和硬件、电路、底层打交道，想要稳定入行 | 嵌入式 / 硬件 | 一个 RTOS（FreeRTOS）、一块更复杂的板子、熟练用示波器/逻辑分析仪调试 |
| 对大模型、对话式产品最有热情，愿意补 CS 底子 | AI 应用 / Agent | Python 工程化、RAG 实战、读几个开源 Agent 框架源码、自己跑通一个带工具调用的 Agent |
| **两边都想要，喜欢「系统打通」的感觉** | **嵌入式 + AI 复合**（推荐） | 把毕设的端云链路讲到能画架构图、能答「为什么保护放边缘、规划放云端」；补一点 Linux/嵌入式 Linux 会更通 |
| 喜欢联网设备、智能家居这类产品 | 物联网 / IoT | 一个云平台（阿里云 IoT/腾讯连连）、BLE 或更多通信协议、一次完整的端到端产品联调 |

**一句话**：方向没有对错，关键是**选定一个，然后让简历重心和它对齐**。先用复合方向的简历去投，面试中观察 HR 和面试官更关心哪一端，再随时微调重心——简历是活的，不是定稿就不动了。
